搭建clojure开发环境

发表于 2015-05-10   |  

操作系统为：Mac OS X 10.10.3

IDE：IntelliJ IDEA 14.1.1

leiningen

Leiningen是你的主要工具， 它用于：

  1. 启动一个 REPL
  2. 下载+安装类库
  3. 运行你的程序
  4. 启动一个服务器， 运行你所写的webapps

安装

brew install leiningen

打开repl

lein repl

运行时报错：

需要执行：sudo scutil --set HostName MyName.local

详见：https://github.com/technomancy/leiningen/issues/1321

创建项目

lein new groupId/artifactId会在当前目录下，生成一个名为“artifactId”的项目，其中groupId和artifactId和Maven里的概念一致。

Idea

添加La Clojure插件

打开preferences 》Plugins 》 Browse Repositories， 搜索Clojure， 下载La Clojure。

新建项目

打开File 》New 》 Project… 》 Intellij Platform Plugin，选中Clojure，一路Next。

如果直接打开刚才用lein命令创建的项目，需要在打开的项目上右单击，点击Add Framework Support…，左侧选中Clojure，点击ok。

使用hexo搭建静态博客

发表于 2015-05-09   |  

Hexo 是一个简单地、轻量地、基于Node的一个静态博客框架。通过Hexo我们可以快速创建自己的博客，然后部署在github pages搭建的免费无限流量的个人博客。

官方文档

Hexo：http://hexo.io/docs/

GitHub Pageshttps://help.github.com/articles/what-are-github-pages/

hexo使用

依赖

  1. Node.js 官网：http://nodejs.org/
  2. Git 官网：http://git-scm.com/

安装

npm install -g hexo-cli

创建博客

hexo init blog

会在当前目录下创建一个名为“blog”的文件夹，

安装依赖

cd blog

npm install

进入博客目录，由于hexo采用node.js开发，安装所有依赖

创建文章

hexo new 文章标题

创建一个标题为“文章标题”的博客

编辑文章

直接打开博客目录下source/_posts/文章标题.md文件进行编辑，比如：

本地启动

hexo server

启动hexo服务器，通过http://localhost:4000/访问

草稿

hexo new draft 草稿标题

对于暂时不想公开的文件，可以放草稿里，本地强行预览草稿可通过命令hexo server --drafts

查看帮助

hexo 或者 hexo help

部署到github

  1. github上创建一个“你的用户名.github.io”的项目，例：yourname.github.io.git

  2. 修改博客目录下全局配置文件：_config.yml，找到deploy或添加：

  1. 执行部署命令：hexo deploy

  2. 用浏览器打开http://yourname.github.io 访问博客

域名绑定

  1. 域名解析中添加一条cname记录指向yourname.github.io

  2. 在博客目录source文件夹下创建一个名为“CNAME”的文件，内容为你的域名，重新部署博客

官方文档：https://help.github.com/articles/setting-up-a-custom-domain-with-github-pages/
