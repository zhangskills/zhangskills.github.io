Spring Boot采用无parent pom方式注意事项

发表于 2015-07-10   |  

Spring Boot默认采用继承spring-boot-starter-parent方式，然后声明一个或多个“Starter POMs”依赖，这种方式使用起来非常方便。

但不是每个人都喜欢继承spring-boot-starter-parent POM。你可能需要使用公司标准parent，或你可能倾向于显式声明所有Maven配置。

使用没有父POM的Spring Boot

如果你不使用spring-boot-starter-parent，通过使用一个scope=import的依赖，你仍能获取到依赖管理的好处

改变Java版本

Spring Boot默认编译级别为Java 1.6，建议使用最新的Java版本

添加Spring Boot Maven插件

Spring Boot包含一个Maven插件，它可以将项目打包成一个可执行jar或war包

编译war时可能遇到的问题

1. 为当前内置的web容器依赖添加provided，防止内置web容器jar生成到WEB-INF/lib目录下

2. 如果采用无父POM的Spring Boot、且无web.xml方式，需要添加war默认插件配置（参考自：stackoverflow）
